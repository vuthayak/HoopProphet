  <action>
    Create the backend API endpoints to serve backtest data and the frontend BacktestPage with all its sub-components.

    A) Backend — Create `server/api/backtest.py`:
       - FastAPI router with `prefix="/api", tags=["backtest"]`
       - Read backtest data from JSON files in `server/data/backtest_logs/`
       - Three endpoints:
         ```python
         GET /api/backtest/summary  → { n_total_predictions, overall_accuracy, brier_score, roi, model_loaded: bool }
         GET /api/backtest/seasons   → [{ season, n_games, accuracy, brier_score, roi }]
         GET /api/backtest/calibration → [{ predicted_bin, predicted_pct, observed_pct, n_predictions }]
         ```
       - Summary endpoint: finds most recent backtest_metrics JSON file, extracts metadata and fold_metrics to compute overall accuracy (weighted average), Brier score (weighted), and ROI (using vig-adjusted breakeven threshold from metadata)
       - Seasons endpoint: reads season_breakdown from same JSON, computes ROI per season using breakeven threshold
       - Calibration endpoint: reads backtest_predictions Parquet file (or a separate calibration JSON if stored), bins predictions into 10% buckets, computes hit rate per bin. If prediction data isn't easily parseable, create a simplified version that generates calibration data from fold_metrics.
       - Error handling: return 404 if no backtest data found, with graceful message
       - Follow the same pattern as server/api/players.py (router, HTTPException, service layer)
       - Add `from server.api.backtest import router as backtest_router` and `app.include_router(backtest_router)` to server/app.py
       - Add the 3 new api methods to hoopprophet/src/api/client.js: getBacktestSummary, getBacktestSeasons, getBacktestCalibration

    B) Frontend — Create `hoopprophet/src/hooks/useBacktest.js`:
       - Three data-fetching states: useBacktestSummary(), useBacktestSeasons(), useBacktestCalibration()
       - Each returns { data, loading, error }
       - Separates fetches so summary can load independently of seasons/calib

    C) Frontend — Create `BacktestSummary.jsx` — Per D-17 and UI-SPEC Backtest wireframe:

       - ROI column color-coded: green for positive, red for negative
       - Header: text-text-secondary, sticky top
       - Rows: hover:bg-bg-card-hover

    E) Frontend — Create `CalibrationChart.jsx` — Per UI-SPEC Data Visualization (Calibration):
       - Uses Visx from @visx/shape (LinePath, Circle), @visx/scale (scaleLinear), @visx/axis, @visx/responsive (ParentSize)
       - Chart type: line path with scatter dots
       - X-axis: Predicted probability 0%–100% (bins of 10%)
       - Y-axis: Observed hit rate 0%–100%
       - Reference line: diagonal at 45° (perfect calibration, dashed, slate-600 color)
       - Data line: blue (#3b82f6) with white dots at each bin
       - Chart area: full content width × 280px tall
       - Gridlines at 20% increments on both axes
       - ParentSize wrapper for responsive sizing (per Pitfall 4)
       - Legend: "Predicted" vs "Observed" in top-right corner
       - Read-only — no interactivity needed per D-17

       - Empty state: "Back-test results will appear here when ready." per Copywriting Contract

    G) Update `App.jsx` to import and render BacktestPage instead of placeholder.
