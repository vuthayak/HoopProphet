  <action>
Inject a runtime SPA routing fallback script into the built dist/index.html. This script runs before React mounts and redirects all non-file-path URLs back to index.html, enabling client-side routing to work with any static file server (serve, nginx, etc.).

After the closing `</body>` tag but before `</html>`, inject:
```html
<script>
(function() {
  var currentPath = window.location.pathname;
  // If path looks like a file request (has extension), let server handle it
  if (currentPath.match(/\.[\w]+$/)) return;
  // If path is root or /index.html, let the SPA handle it
  if (currentPath === '/' || currentPath === '/index.html') return;
  // Otherwise (e.g., /backtest, /player/123), redirect to index.html preserving path
  window.location.href = '/?redirect=' + encodeURIComponent(currentPath);
})();
</script>
```

Also add a corresponding entry in App.jsx's useEffect or a dedicated useEffect in main.jsx to handle the redirect parameter:
```javascript
// In App.jsx or a useEffect in main.jsx, check for redirect param:
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const redirect = params.get('redirect');
  if (redirect) {
    // Replace URL without the redirect param, then let React Router handle it
    window.history.replaceState(null, '', redirect);
  }
}, []);
```

Actually, a simpler approach: instead of the index.html redirect, use serve's `--single` flag which is designed exactly for this purpose. Check if `serve -s dist -l 3000 --single` works with the current Dockerfile, and if so, update the CMD accordingly. The `--single` flag makes serve redirect all requests for missing files to index.html.

```
CMD ["serve", "-s", "dist", "-l", "3000"]
```

To:
```

```

The `--single` flag makes serve redirect all requests for missing files back to index.html, enabling client-side routing to work correctly. This is the standard way to deploy SPAs with the `serve` package.
