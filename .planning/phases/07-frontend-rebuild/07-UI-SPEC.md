# Phase 7 — UI Design Contract

> Visual and interaction contract for the HoopProphet frontend rebuild. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind CSS design system per D-04) |
| Preset | not applicable |
| Component library | none (custom components built with Tailwind utility classes) |
| Icon library | Lucide React (lightweight, tree-shakeable, consistent 24px grid) |
| Chart library | Visx (Airbnb's D3 wrapper — per D-06, separate bar charts per prop card) |
| Font | Inter (body, labels) via Google Fonts CDN + system font stack fallback |
| Routing | React Router v7 (SPA client-side per D-01) |
| Build | Vite (per D-21, migrating from CRA) |
| Animation | CSS transitions + Tailwind animate utilities (per D-09, polished not flashy) |
| Testing | React Testing Library (per D-23) |

### Rationale

MUI is removed entirely (D-04). The project builds a custom design system with Tailwind CSS for dark-mode-native, data-dashboard aesthetics. No shadcn initialized because the project uses no Radix primitives — all components are custom Tailwind compositions. Visx is chosen for charts because it provides low-level D3 access for the hit-rate bar chart requirement (D-06, UI-08) while integrating cleanly with React. Lucide React is a standard lightweight icon set that pairs well with Tailwind dark themes.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | p-1 | 4px | Icon gaps, inline badge padding |
| sm | p-2 | 8px | Compact element spacing, badge margins |
| md | p-4 | 16px | Default element spacing, card padding |
| lg | p-6 | 24px | Section padding, card margins |
| xl | p-8 | 32px | Layout grid gaps, page section spacing |
| 2xl | p-12 | 48px | Major section breaks |
| 3xl | p-16 | 64px | Page-level top/bottom padding |

Exceptions:
- **44px touch targets** for interactive elements on mobile (meets WCAG 2.1 SC 4.1.2)
- **Half-point slider thumb:** 20px × 20px slider thumb for precise 0.5 increment control (D-08)
- **Prop card minimum width:** 320px to prevent content overflow on data-dense cards

## Typography

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Display (probability badge) | 28px | 700 (bold) | 1.0 | `text-[28px] font-bold leading-none` |
| Heading (page titles, section headers) | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Body (default text, table cells, descriptions) | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-relaxed` |
| Label (stat names, section tabs, axis labels) | 12px | 500 (medium) | 1.2 | `text-xs font-medium leading-tight` |

### Font Stack

```css
font-family: 'Inter', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

Loaded via `<link>` in `index.html` — `weights: 400;500;600;700`. No local font files needed.

### Type Decisions Rationale

- **4 sizes only:** Probability badge needs visual dominance (28px bold); page structure needs headings (20px semibold); reading text needs comfort (14px/1.5); data labels need scannability (12px medium). No intermediate sizes needed for this data dashboard.
- **14px body, not 16px:** Data-dense dashboards read better at 14px with generous line height (1.5). More information per viewport without squinting. This is standard for analytics tools (Grafana, Linear, Vercel Dashboard).
- **Inter font:** Designed for UI/data, excellent legibility at small sizes, free, widely available via CDN. Includes tabular figures for stat alignment.

### Accent Reserved For

Accent color (`#22c55e` green-500) is reserved for these specific elements ONLY:

1. **High-probability badge background** — probability > 65%
2. **Primary CTA buttons** — "Analyze Player" on home page
3. **Active tab indicator** — underline/highlight on selected tab
4. **Chart "over" indicators** — green bar segments for hit rates above threshold
5. **Success toast icon** — confirmation toasts

Accent secondary (`#eab308` yellow) is reserved for:

1. **Moderate-probability badge background** — probability 40%–65%
2. **QUESTIONABLE injury status badge** — D-12 yellow badge

Accent tertiary (`#ef4444` red) is reserved for:

1. **Low-probability badge background** — probability < 40%
2. **OUT injury status badge** — D-12 red badge
3. **INJURY badge** — D-12 orange variant (`#f97316` orange-500)
4. **Error states and toasts**

## Interaction Patterns

### 1. Player Search (Autocomplete)

| Aspect | Specification |
|--------|---------------|
| Trigger | User types ≥ 2 characters in search input |
| Debounce | 300ms after last keystroke before API call |
| API | `GET /api/players?search={query}&active_only=true` |
| Response | `[{player_id, full_name, position, team_id}]` |
| Dropdown | Shows up to 10 results, each with player name + team position |
| Selection | Click or Enter selects player → navigates to `/player/{id}` |
| Escape | Closes dropdown, clears focus |
| No results | Dropdown shows "No players found" message |
| Mobile | Full-width, no dropdown truncation |

### 2. Probability Badge

| Aspect | Specification |
|--------|---------------|
| Display | Large centered percentage, e.g. "72%" |
| Color thresholds | Green (#22c55e) for > 65%, Yellow (#eab308) for 40–65%, Red (#ef4444) for < 40% |
| Accessibility | Text on badge uses inverted dark text (#0f1117) for contrast ratio > 4.5:1 |
| Direction label | "OVER" displayed below percentage in uppercase label text (12px, weight 500, text-muted) |
| Size | 72px × 72px on desktop, 56px × 56px on mobile |

### 3. Line Slider (Adjustable Lines)

| Aspect | Specification |
|--------|---------------|
| Range | Default line ± 10.0 (e.g., default 24.5 → range 14.5 to 34.5) |
| Step | 0.5 increments (sportsbook standard — D-08) |
| Display | Current line value shown below slider thumb, e.g. "24.5" |
| Behavior | On release, calls `GET /api/players/{id}/hitrates?stat={stat}` with new line value |
| Visual | Tailwind range input with accent track fill (blue-500), dark track (slate-700) |
| Thumb | 20px × 20px white circle with subtle shadow |
| Label | Stat name displayed above slider, e.g. "Points" |

### 4. Hit Rate Chart (per Prop Card)

| Aspect | Specification |
|--------|---------------|
| Library | Visx (BarStack or BarGroup) |
| Bars | 4 bars: L5, L10, L20, Season — always in that order |
| Bar height | Proportional to rate (0%–100%) |
| Bar width | Uniform, group of 4 bars spanning ~200px |
| Labels | Rate percentage above each bar, sample count below ("3/5") |
| Colors | Green (#22c55e) fill for rates ≥ 55%, Yellow (#eab308) for 40–55%, Red (#ef4444) for < 40% |
| Null handling | Window with insufficient data (< 5 games) shows "—" instead of bar |
| Animation | Bars animate in from baseline on mount (200ms ease-out) |
| Reactive | Re-animates when line slider changes and new hit rates arrive |

### 5. Alert Badges (Injury/News)

| Alert Type | Badge Color | Text | Hover |

| OUT | `bg-red-500 text-white` | "OUT" | Tooltip: "{headline}" |

## Data Flow & API Integration

### Page-Level Fetch Strategy (per D-16)

| Page | Parallel Fetches | Endpoint |
|------|-----------------|----------|
| Home | None (search is on-demand) | — |
| Player Overview | Player detail, Props, Alerts (Promise.all) | `/api/players/{id}`, `/api/players/{id}/props`, `/api/players/{id}` (alerts embedded) |
| Player Game Logs | (already fetched on page load) | `/api/players/{id}/gamelogs` |
| Player News | (already fetched on page load) | `/api/players/{id}/news` |
| Backtest | Summary, Season breakdown, Calibration | `/api/backtest/summary`, `/api/backtest/seasons`, `/api/backtest/calibration` (endpoints TBD in Phase 8) |
| Line Slider change | Hit rates for specific stat | `/api/players/{id}/hitrates?stat={stat}` (with adjusted line) |

**Note:** Hit rates refetch only when line slider is adjusted (D-16). All other data fetched once on page load.

### API Health Check

On app mount, fetch `/api/health`. If `model_loaded: false`, show a subtle banner: "Predictions unavailable — model is reloading." All other features still work. (D-10 graceful degradation.)

## Data Visualization Specifications

### Visx Bar Chart — Hit Rate per Prop

**Configuration:**

```
- Chart type: BarGroup (4 bars per group)
- X-axis: Window labels ["L5", "L10", "L20", "Season"]
- Y-axis: Rate 0%–100%
- Bar colors: Conditional on rate value (green/yellow/red per thresholds)
- Bar width: 28px with 8px gap
- Chart area: ~240px wide × 160px tall (fits inside PropCard)

- Axes: Y-axis with gridlines at 25% increments (dashed, slate-700)
         X-axis labels in label font (12px, medium, slate-400)

```
- Chart type: Line path with scatter dots
- X-axis: Predicted probability 0%–100% (bins of 10%)

- Chart area: Full content width × 280px tall
- Axes: Both with gridlines at 20% increments
- Legend: "Predicted" vs "Observed" in top-right

## API Response Shapes (Reference)

These shapes are consumed by the frontend and drive component design:

```
GET /api/players?search={query}
→ [{ player_id, full_name, position, team_id }]

GET /api/players/{id}
→ { player_id, full_name, position, team_id, default_lines: {pts: 24.5, ...}, alerts: [{alert_type, severity, subcategory, last_updated_at}] }

GET /api/players/{id}/props
→ { player_id, default_lines: {...}, top_props: [{stat, line, probability, direction, hit_rates: {L5: {rate, count}, L10: {rate, count}, L20: {rate, count}, season: {rate, count}}}] }

GET /api/players/{id}/hitrates?stat={stat}&line={line}
→ { player_id, stat, line, hit_rates: {L5: {rate, count}, L10: {rate, count}, L20: {rate, count}, season: {rate, count}} }

GET /api/players/{id}/gamelogs?limit=50
→ [{ player_id, game_id, season, game_date, matchup, wl, is_dnp, pts, reb, ast, ... }]

GET /api/players/{id}/news
→ { player_id, alerts: [{alert_type, subcategory, severity, source, headline, updated_ago}], news_items: [...], stale_warning: "..." | null }
```
