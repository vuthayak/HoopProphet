<objective>
Implement contextual game features (rest days, back-to-back, home/away, opponent defense, pace) and historical matchup features (player performance vs specific opponents).

Purpose: Contextual features capture game environment that affects player performance. Matchup history features capture player-specific opponent tendencies. Together they provide the situational context that rolling stats alone cannot — a player's average is different when they play at home vs away, rested vs back-to-back, and against strong vs weak defenses.

Output: Two processor modules (contextual_features.py, matchup_features.py) with comprehensive test coverage.
</objective>

    **A. Create server/pipeline/processors/contextual_features.py:**

    Function signature: `compute_contextual_features(df: pd.DataFrame, team_stats_df: pd.DataFrame, teams_df: pd.DataFrame, players_df: pd.DataFrame) -> pd.DataFrame`

    Input contract:
    - `df`: game logs filtered to is_dnp==0, with columns: player_id, game_id, season, game_date, matchup, and all stat columns. Already sorted by player_id, game_date from Plan 01's rolling features.
    - `team_stats_df`: DataFrame from `get_team_stats_df()` with columns: team_id, season, def_rating, off_rating, net_rating, pace
    - `teams_df`: DataFrame from `get_teams_df()` with columns: team_id, abbreviation, full_name
    - `players_df`: DataFrame from `get_players_df()` with columns: player_id, full_name, position, team_id

    Implementation:

    1. **Rest days & back-to-back (FEAT-04):**
       ```python
       df = df.sort_values(['player_id', 'game_date']).copy()
       df['game_date_dt'] = pd.to_datetime(df['game_date'])
       df['rest_days'] = (
           df.groupby('player_id')['game_date_dt']
           .diff()
           .dt.days
           .fillna(7)
           .clip(upper=14)
       )
       df['is_b2b'] = (df['rest_days'] == 1).astype(int)
       ```
       - fillna(7): first game of season assumes rested
       - clip(upper=14): cap at 14 days (All-Star break / long rest)
       - is_b2b: 1 if games on consecutive days, 0 otherwise

    2. **Home/away indicator (FEAT-05):**
       ```python
       df['is_home'] = df['matchup'].str.contains(' vs. ', regex=False).astype(int)
       ```
       Matchup format: "DEN vs. LAL" = home (1), "DEN @ LAL" = away (0)

    3. **Opponent team extraction:**
       ```python
       def _extract_opponent(matchup: str) -> str:
           if ' vs. ' in matchup:
               return matchup.split(' vs. ')[1].strip()
           elif ' @ ' in matchup:
               return matchup.split(' @ ')[1].strip()
           return None
       ```
       Apply: `df['opp_abbr'] = df['matchup'].apply(_extract_opponent)`
       Map to team_id: `abbr_to_id = dict(zip(teams_df['abbreviation'], teams_df['team_id']))`
       `df['opp_team_id'] = df['opp_abbr'].map(abbr_to_id)`

    4. **Player's team extraction (needed for team pace join):**
       ```python
       df['player_team_abbr'] = df['matchup'].str.split(r' vs\. | @ ', regex=True).str[0].str.strip()
       df['player_team_id'] = df['player_team_abbr'].map(abbr_to_id)
       ```

    5. **Opponent DEF_RATING and opponent pace (FEAT-03, FEAT-06):**
       ```python
       opp_stats = team_stats_df[['team_id', 'season', 'def_rating', 'pace']].rename(
           columns={'team_id': 'opp_team_id', 'def_rating': 'opp_def_rating', 'pace': 'opp_pace'}
       )
       df = df.merge(opp_stats, on=['opp_team_id', 'season'], how='left')
       ```
       Per D-08: using team-level DEF_RATING (no per-position data). Player position is added as a separate feature so the model can learn position×defense interactions.

    6. **Player's team pace (FEAT-06):**
       ```python
       team_pace = team_stats_df[['team_id', 'season', 'pace']].rename(
           columns={'team_id': 'player_team_id', 'pace': 'team_pace'}
       )
       df = df.merge(team_pace, on=['player_team_id', 'season'], how='left')
       ```

    7. **Player position (D-08 position proxy):**
       ```python
       position_map = players_df[['player_id', 'position']]
       df = df.merge(position_map, on='player_id', how='left')
       ```
       Keep position as a string column — LightGBM will handle it as categorical in Phase 3.

    8. Drop intermediate columns not needed downstream: `df.drop(columns=['game_date_dt', 'player_team_abbr'], inplace=True)`

    Return df with new columns: rest_days, is_b2b, is_home, opp_abbr, opp_team_id, player_team_id, opp_def_rating, opp_pace, team_pace, position.

    Use `logging.getLogger(__name__)` for module logging.

    **B. Create server/pipeline/processors/matchup_features.py:**

    Function signature: `compute_matchup_features(df: pd.DataFrame, all_game_logs: pd.DataFrame) -> pd.DataFrame`

    Input contract:
    - `df`: game logs with contextual features applied (has opp_team_id column from contextual_features.py). One row per player per played game.
    - `all_game_logs`: FULL game logs DataFrame (including all seasons, is_dnp==0 filtered). Used as the history source.

    Per D-07: use last 2 seasons of matchup data only.

    Implementation:

    1. Define previous-season mapping for 2-season window:
       ```python
       PREV_SEASON = {
           "2020-21": None,
           "2021-22": "2020-21",
           "2022-23": "2021-22",
           "2023-24": "2022-23",
           "2024-25": "2023-24",
       }
       ```

    2. For each stat in PRIMARY_STATS, create a `matchup_avg_{stat}` column initialized to NaN.

    3. Vectorized approach for performance:
       - Join df with all_game_logs to find prior games against the same opponent
       - For each row in df, the matchup history includes games from `all_game_logs` where:
         - Same player_id
         - Same opponent (opp_team_id matches — need to extract opponent from all_game_logs matchup too)
         - Season is current season OR the previous season (2-season window per D-07)
         - game_date is strictly BEFORE current game's game_date
         - is_dnp == 0

       Efficient implementation approach:
       a. Add opp_team_id to all_game_logs using the same _extract_opponent logic (import or duplicate the helper).
       b. For each (player_id, opp_team_id, game_date, season) in df:
          - Filter all_game_logs to matching player+opponent within [current_season, prev_season], game_date < current_game_date
          - Compute mean of PRIMARY_STATS
       c. Use a merge-based approach for efficiency:
          - Cross-reference: merge df[['player_id', 'game_id', 'opp_team_id', 'season', 'game_date']] with all_game_logs on player_id + opp_team_id
          - Filter to 2-season window and game_date < current_game_date
          - Groupby original game_id, compute mean of stats
          - Join back to df

    4. If no prior matchup games exist for a (player, opponent), matchup_avg columns remain NaN. LightGBM handles NaN natively.

    5. Use tqdm for progress logging if implementation iterates per-player.

    Return df with new columns: matchup_avg_pts, matchup_avg_reb, matchup_avg_ast, matchup_avg_stl, matchup_avg_blk, matchup_avg_fg3m, matchup_avg_plus_minus, matchup_avg_min (one per PRIMARY_STAT).
