    Create server/pipeline/processors/target_generator.py.

    **Function: `generate_targets(wide_df: pd.DataFrame) -> pd.DataFrame`**

    Transforms the wide-format feature matrix (one row per player×game, with rolling + contextual + matchup features) into long format (one row per player×game×stat_type×line_value) with binary `hit` column. Per D-03 (multiple lines) and D-04 (median-centered).

    Implementation:

    1. Import: `from server.pipeline.feature_config import ALL_TARGET_STATS, STAT_TYPE_MAP, COMBO_STATS, N_THRESHOLD_LINES, PRIMARY_STATS`

    2. **Identify columns to carry forward into long format:**
       - ID columns: `['player_id', 'game_id', 'season', 'game_date']`
       - Feature columns: all columns matching patterns `_avg_L`, `_std_L`, `_season_avg`, `games_played_season`, `rest_days`, `is_b2b`, `is_home`, `opp_def_rating`, `opp_pace`, `team_pace`, `position`, `matchup_avg_`
       - Exclude: raw stat columns (pts, reb, ast, ...), intermediate columns (opp_abbr, opp_team_id, player_team_id, matchup, wl, game_date_dt)
       - Build feature_cols list dynamically:
         ```python
         id_cols = ['player_id', 'game_id', 'season', 'game_date']
         exclude_patterns = set(STAT_COLS) | set(COMBO_STATS.keys()) | {
             'matchup', 'wl', 'is_dnp', 'opp_abbr', 'opp_team_id',
             'player_team_id', 'game_date_dt',
         }
         feature_cols = [c for c in wide_df.columns
                         if c not in id_cols and c not in exclude_patterns]
         ```
         (STAT_COLS and combo raw columns are excluded because they would leak the actual game outcome — the rolling/expanding features derived from them with .shift(1) are the safe versions)

    3. **Compute center lines per player per target stat:**
       For each stat in ALL_TARGET_STATS:
       a. Get the raw stat column: for base stats (pts, reb, etc.) use the column directly from wide_df. For combo stats (pra, pa, pr), the combo column was already computed in rolling_features.py and is present in wide_df.
       b. Compute L20 rolling median with .shift(1) for line derivation:
          ```python
          wide_df[f'_line_median_{stat}'] = (
              wide_df.groupby('player_id')[stat]
              .rolling(20, min_periods=5)
              .median()
              .reset_index(level=0, drop=True)
          )
          wide_df[f'_line_median_{stat}'] = (
              wide_df.groupby('player_id')[f'_line_median_{stat}'].shift(1)
          )
          ```
       c. Round to nearest 0.5: `round(median * 2) / 2`

    4. **Generate long-format rows:**
       ```python
       long_frames = []
       for stat in ALL_TARGET_STATS:
           stat_df = wide_df.dropna(subset=[f'_line_median_{stat}']).copy()
           base_line = (stat_df[f'_line_median_{stat}'] * 2).round() / 2

           for offset in range(-(N_THRESHOLD_LINES // 2), (N_THRESHOLD_LINES // 2) + 1):
               line_val = (base_line + offset).clip(lower=0.5)
               row_df = stat_df[id_cols + feature_cols].copy()
               row_df['stat_type'] = STAT_TYPE_MAP[stat]
               row_df['line_value'] = line_val
               row_df['hit'] = (stat_df[stat] > line_val).astype(int)
               long_frames.append(row_df)

       long_df = pd.concat(long_frames, ignore_index=True)
       ```

       Per N_THRESHOLD_LINES=3: offsets are [-1, 0, +1], producing 3 lines per stat. Lines floored at 0.5 (no zero/negative lines).

    5. **Clean up:**
       Drop temporary `_line_median_*` columns from wide_df (don't mutate caller's DataFrame — use .copy() earlier or drop after concat).
       Ensure stat_type is integer dtype.
       Ensure hit is integer dtype (0 or 1).

    6. Return long_df.

    **Row count estimate:** ~22 players × ~80 games × 10 stat_types × 3 lines = ~52K rows from fixture; production: ~450 players × ~80 games × 10 × 3 = ~10.8M rows.

    Use `logging.getLogger(__name__)` for logging. Log total rows generated and stat_type counts.
