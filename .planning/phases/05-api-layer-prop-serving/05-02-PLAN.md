<objective>
Build the computation layer for prop serving: hit rates from game log history, default stat lines derived from recent performance, model-based probability prediction, and top-prop ranking.

Purpose: This is the core business logic that the API endpoints in Plan 03 will expose. Hit rates answer "how often does this player hit over X?" for multiple windows (PROP-01). Default lines answer "what's a reasonable line?" (PROP-02). Model prediction provides calibrated probability (PROP-05). Top-prop ranking surfaces the 4-5 best props (PROP-04).

Output: HitRateService and PredictionService with full test coverage, ready to be wired into API endpoints.
</objective>

    - Test 2: compute_hit_rates for a player with <5 games in L5 window returns None for that window (not enough data)
    - Test 3: get_default_lines derives median-based lines from recent game logs, rounded to 0.5 increments (per feature_config.py target generation logic using N_THRESHOLD_LINES=3)

  </behavior>
  <action>
    Create `server/services/hitrate_service.py` with HitRateService class:

    **Constructor:** Takes `sqlite3.Connection` (injected, same pattern as PlayerService).

    **Method `get_recent_games(player_id: int, n: int = 20) -> list[dict]`:**
    - Queries game logs for the player using `get_game_logs_df(self.conn)`, filters `is_dnp == 0`
    - Sorts by `game_date` desc, returns the N most recent rows as list of dicts
    - Dict keys: player_id, game_id, season, game_date, matchup, wl, min, pts, reb, ast, stl, blk, fg3m, fgm, fga, ftm, fta, oreb, dreb, tov, pf, plus_minus

    **Method `compute_hit_rates(player_id: int, line_value: float, stat: str) -> dict`:**
    - Gets all non-DNP game logs for the player
    - Computes `(actual_stat > line_value)` for each game
    - For each window in WINDOWS_PRIMARY (L5=5, L10=10, L20=20) and full season:
      - Slice last N games (or entire current season for season window)
      - `hit_rate = count(actual > line) / total_games`
      - If window has fewer than 5 games, set rate to `None` (insufficient sample)
    - Returns: `{"L5": {"rate": float|None, "count": int}, "L10": {...}, "L20": {...}, "season": {...}}`
    - This satisfies PROP-01: hit rates across L5, L10, L20, season windows

    **Method `get_default_lines(player_id: int) -> dict`:**
    - Gets all non-DNP game logs for the player
    - For each stat in PRIMARY_STATS (and combo stats via COMBO_STATS):
      - Compute rolling median of the stat over the last 20 games (min 5 games required)
      - Round to 0.5 increments: `round(median * 2) / 2`
      - If fewer than 5 games, skip that stat (insufficient data)
    - For combo stats (pra, pa, pr): compute median of the sum of component stats
    - Returns: `{"pts": 24.5, "reb": 8.5, ...}` — dict mapping stat name to default line value
    - This satisfies PROP-02: default stat lines derived from player's recent performance

    **Method `get_top_stats(player_id: int) -> list[str]`:**
    - Returns the stats for which the player has enough data (≥5 game logs with >0 values)
    - Filters out stats where the player averages <1.0 per game (low-volume, irrelevant props)
    - Selects top stats by variance/game count (the stats a player is "known for")
    - Returns up to 5 stat names sorted by relevance (volume × variance)
    - This is the foundation for PROP-04 (top props per player)

    Write tests in `server/tests/test_hitrate_service.py` using synthetic game log data inserted into an in-memory SQLite database. Cover all 6 behavior specs.

  <action>
    Create `server/services/prediction_service.py` with PredictionService class:

    **Constructor:** Takes `model_artifact: Dict[str, Any] | None` and `sqlite3.Connection`. The model artifact is the loaded dict from `load_artifact()` passed from app.state during FastAPI dependency injection.

    **Method `predict_prop(player_id: int, stat: str, line_value: float) -> float | None`:**
    - If `self.model_artifact is None`, return `None` (model not loaded, graceful degradation)
    - Build a single-row feature DataFrame matching `artifact['feature_columns']`
    - The feature row must be constructed from the player's most recent game log data. Key features include rolling averages (L5, L10, L20), opponent defensive rating, rest days, home/away, etc. However, for serving, we use the **most recent row from the features Parquet** for this player, filtered to the matching stat_type.
    - Alternative approach (simpler, more robust): Load the features.parquet, filter to the player's most recent game for each stat_type, and use those pre-computed features. This avoids recomputing features on-the-fly (per PITFALLS.md: "Computing features on-the-fly per prediction request" is an anti-pattern). The feature store is the source of truth.
    - Set `stat_type` to the integer code from `STAT_TYPE_MAP[stat]`
    - Set `line_value` to the provided line
    - Call `predict_proba(self.model_artifact, X)` to get the calibrated probability
    - Returns: probability as float (e.g., 0.72)
    - This satisfies PROP-05: ML-predicted probability of hitting over the line

    **Method `get_top_props(player_id: int, max_props: int = 5) -> list[dict]`:**
    - If `self.model_artifact is None`, return `[]` (no predictions without model)
    - Get default lines from `HitRateService.get_default_lines(player_id)`
    - Get top stats from `HitRateService.get_top_stats(player_id)`
    - For each top stat (up to `max_props`):
      - Get default line value from the lines dict
      - Call `self.predict_prop(player_id, stat, line_value)` for probability
      - Call `self.hitrate_service.compute_hit_rates(player_id, line_value, stat)` for hit rates
      - Build prop dict: `{"stat": stat, "line": line_value, "probability": float, "direction": "over" if prob > 0.5 else "under", "hit_rates": {...}}`
    - Sort by probability descending
    - Returns: list of prop dicts (up to 5)
    - This satisfies PROP-04: top 4-5 props per player ranked by probability

    **Method `get_prop_analysis(player_id: int, stat: str, line_value: float) -> dict`:**
    - Get probability from `predict_prop(player_id, stat, line_value)`
    - Get hit rates from `self.hitrate_service.compute_hit_rates(player_id, line_value, stat)`
    - Get default line from `self.hitrate_service.get_default_lines(player_id)` for this stat
    - Returns: `{"stat": stat, "line": line_value, "default_line": float, "probability": float, "direction": str, "hit_rates": dict, "sample_sizes": dict}`
    - This is the full prop analysis view for PROP-01 + PROP-02 + PROP-05

    **Dependency:** PredictionService depends on HitRateService (injected via constructor or created internally with the shared DB connection).

    - In-memory SQLite with seeded game log data
    - Cover all 6 behavior specs
