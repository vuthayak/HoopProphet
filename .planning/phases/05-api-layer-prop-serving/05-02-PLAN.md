    **1. Create server/services/hitrate_service.py** — Hit rate computation per D-09:

    Imports:
    ```python
    import logging
    from server.services.player_service import get_player_game_logs
    from server.pipeline.feature_config import ALL_TARGET_STATS, WINDOWS_PRIMARY

    logger = logging.getLogger(__name__)
    ```

    Key function:
    ```python
    def get_hit_rates(player_id: int, stat: str, seasons: list[str] | None = None) -> dict:
    ```
    - Calls `get_player_game_logs(player_id, seasons=seasons, limit=None)` to get all relevant game logs
    - Filters to non-DNP games (is_dnp == 0) and games where the stat value is not null/NaN
    - Computes hit rate for each window in WINDOWS_PRIMARY (5, 10, 20) plus full season:
      - For L5: take last 5 non-DNP games, count how many had stat > 0 (or any threshold), compute rate as hits/count. But per CONTEXT.md D-09, hit rate is specifically "over the default line" — so hit rate needs a line value.
      - IMPORTANT CORRECTION: Hit rate is percentage of games where a player's stat exceeds the default line for that stat. The line comes from `get_default_lines`.
      - So `get_hit_rates` needs a `line_value: float` parameter.
      - For each window: count games where stat > line_value, divide by total games in window.
      - Return format per D-09: `{L5: {rate: 0.8, count: 5}, L10: {rate: 0.7, count: 10}, L20: {rate: 0.65, count: 20}, season: {rate: 0.68, count: 55}}`
      - For windows with <5 games, return None per D-11
    - Raises ValueError with "Player not found: {player_id}" if player has no game logs

    Implement these functions:

    **`get_default_lines(player_id: int, seasons: list[str] | None = None) -> dict[str, float]`** per D-01, D-02, D-03:
    - Get game logs via get_player_game_logs(player_id, seasons=seasons)
    - Filter to non-DNP games (is_dnp == 0)
    - For each PRIMARY_STAT (pts, reb, ast, stl, blk, fg3m, min): compute median of last 20 non-DNP games, rounded to nearest 0.5 per D-01
    - For combo stats (pra, pa, pr): compute per D-03 — for each game, sum the component stats (e.g., pts+reb+ast for PRA), then take median of last 20 sums, rounded to nearest 0.5
    - Filter out stats where player averages <1.0 per game per D-04
    - Require minimum 5 games per D-02 — if fewer than 5 non-DNP games, exclude that stat from default lines
    - Return dict: `{"pts": 24.5, "reb": 10.0, ...}` with only stats that pass both filters

    **`get_predictions(player_id: int, model_artifact: dict | None) -> list[dict]`** per D-13, D-14, D-08:
    - If model_artifact is None, return empty list per D-10
    - Read features.parquet via pd.read_parquet(PARQUET_PATH)
    - Filter to player_id rows. Sort by game_date descending. Take the most recent row per stat_type per D-13.
    - If no rows found for player, return empty list per D-14
    - For each stat in the player's default lines:
      - Get the feature row for that stat_type
      - Call predict_proba(model_artifact, feature_row_df)
      - Round probability to nearest 1% (0.01) per D-08
      - Return dict with: stat, line (from default_lines), probability (rounded), direction ("over" or "under")
    - The direction is "over" because the model predicts P(stat > line)

    **`get_top_props(player_id: int, model_artifact: dict | None, n: int = 5) -> list[dict]`** per D-05, D-06:
    - Get all predictions via get_predictions
    - Sort by probability descending per D-05
    - Return top n props (max 5 per D-06)
    - Filter out stats where player averages <1.0 per game (already handled in get_default_lines)

    **`get_player_props(player_id: int, model_artifact: dict | None, seasons: list[str] | None = None) -> dict`** per D-07:
    - Get default_lines via get_default_lines
    - Get predictions via get_predictions
    - Get top_props via get_top_props
    - For each prop in top_props, add hit_rates by calling get_hit_rates with that stat and line
    - Return lean JSON per D-07:
    ```python
    {
        "player_id": player_id,
        "default_lines": default_lines,
        "top_props": [
            {
                "stat": "pts",
                "line": 24.5,
                "probability": 0.72,
                "direction": "over",
                "hit_rates": {
                    "L5": {"rate": 0.8, "count": 5},
                    "L10": {"rate": 0.7, "count": 10},
                    "L20": {"rate": 0.65, "count": 20},
                    "season": {"rate": 0.68, "count": 55}
                }
            },
            ...
        ]
    }
    ```

    **Helper function: `_round_half(value: float) -> float`:** Round to nearest 0.5, e.g., 24.3 → 24.5, 24.7 → 24.5, 25.0 → 25.0
    **Helper function: `_round_percent(value: float) -> float`:** Round to nearest 0.01 for probability display per D-08

    **2. Create server/api/players.py** — FastAPI router for player endpoints per D-15 (flat /api prefix, resource-based routes):

    ```python
    from fastapi import APIRouter, HTTPException, Request
    from server.services.player_service import get_players, search_players, get_player_by_id, get_player_game_logs
    from server.services.prediction_service import get_player_props, get_default_lines, get_top_props, get_predictions
    from server.services.hitrate_service import get_hit_rates

    router = APIRouter(prefix="/api", tags=["players"])
    ```

    Endpoints:
    - `GET /api/players` — List all players (active only by default). Query params: `search: str | None`, `active_only: bool = True`. Calls `search_players(query)` if search provided, otherwise `get_players(active_only)`. Returns lean list per D-07: `[{"player_id": ..., "full_name": ..., "position": ..., "team_id": ...}, ...]`

    - `GET /api/players/{player_id}` — Single player detail. Returns player info merged with default_lines. Calls `get_player_by_id(player_id)` and `get_default_lines(player_id)`. Raises 404 if player not found per D-12.

    - `GET /api/players/{player_id}/props` — Top props with hit rates. Gets model_artifact from `request.app.state.model_artifact`. Calls `get_player_props(player_id, model_artifact)`. Returns the combined dict per D-07. Model artifact may be None (graceful degradation per D-10 — still returns 200 with empty top_props).

    - `GET /api/players/{player_id}/gamelogs` — Recent game logs. Query params: `limit: int = 50`, `seasons: str | None = None` (comma-separated). Calls `get_player_game_logs(player_id, seasons, limit)`. Returns lean list of game log dicts per D-07. Raises 404 if player not found per D-12.

    - `GET /api/players/{player_id}/hitrates` — Hit rates for a specific stat. Query params: `stat: str` (required), `seasons: str | None = None`. Calls `get_hit_rates(player_id, stat, seasons)`. Note: hit rates require a line value — use the default line for that stat from `get_default_lines`. Returns `{stat: ..., line: ..., hit_rates: {L5: ..., L10: ..., L20: ..., season: ...}}`.

    - `GET /api/players/{player_id}/lines` — Default lines only. Calls `get_default_lines(player_id)`. Returns `{"player_id": ..., "lines": {"pts": 24.5, ...}}`.

    Each endpoint catches ValueError from service layer and re-raises as HTTPException with 404 and detail "Player not found" per D-12.

| Boundary | Description |
|----------|-------------|
| client → API endpoints | Untrusted query params (player_id, stat, search, limit, seasons). Must validate and sanitize. |
| API → features.parquet | Trusted local file read. Model reads pre-computed features — no user-controlled data path. |
| API → model artifact | Trusted local file loaded at startup. Not user-controllable. |
