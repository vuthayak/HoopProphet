<objective>
Refactor FastAPI app to serve player/team data from SQLite cache and pre-load the trained model artifact at startup — eliminating live NBA API calls and per-request model training (per CLNP-02, CLNP-03). Also expose recent game logs for any player (per PROP-06).

Purpose: Replace V1's broken patterns (live nba_api calls, per-request model training, Gemini dependency) with V2 patterns (SQLite cache, model artifact preloading, structured service layer). This is the foundation all other Phase 5 endpoints depend on.

Output: Refactored FastAPI app with service layer, config module, player/team/game-log endpoints reading from cache, and model artifact preloading at startup.
</objective>

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted HTTP input (search queries, player_id params) |
| API → SQLite | Local trusted store, no external network calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Client→API endpoint params | mitigate | Validate player_id as positive int, query strings as length-bounded strings; use Pydantic models for all request/response schemas |
| T-05-02 | Information Disclosure | Game log endpoint | accept | No PII in game logs; data is public NBA stats |
| T-05-03 | Denial of Service | Search endpoint | accept | SQLite reads are fast; no external API calls in request path eliminates primary DoS vector |
| T-05-04 | Tampering | Model artifact path | mitigate | Config resolves path from env var with fixed default; no user input in path construction |
</threat_model>
