<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted HTTP input (player_id, stat, line_value) |
| API → Services | Trusted internal calls (validated inputs) |
| Services → Model artifact | Trusted local file |
| Services → SQLite/Parquet | Trusted local data store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | Tampering | player_id, stat, line_value params | mitigate | Pydantic models validate types; player_id as positive int, stat against ALL_TARGET_STATS enum, line_value as positive float |
| T-05-09 | Elevation of Privilege | Model artifact access | accept | Single artifact, loaded at startup, no per-request path manipulation |
| T-05-10 | Information Disclosure | Probability precision in API responses | mitigate | Round all probability values to 2 decimal places (1% precision) to prevent model reconstruction attacks per PITFALLS.md |
| T-05-11 | Denial of Service | Unbounded queries on hitrate/probability endpoints | mitigate | Service methods limit computation; no on-the-fly feature engineering (all pre-computed) |
</threat_model>
