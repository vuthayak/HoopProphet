    Function `_get_player_team_tenure(conn: sqlite3.Connection, player_id: int, season: str) -> dict[int, tuple[str, str]]`:
    - Query player_game_logs for this player and season: `SELECT game_date, matchup FROM player_game_logs WHERE player_id = ? AND season = ? AND is_dnp = 0 ORDER BY game_date`
    - Parse the MATCHUP column to extract team abbreviation: matchup format is "TEA vs. OPP" or "TEA @ OPP" — the first 3 characters (before the space) are the player's team
    - Group consecutive games by team abbreviation
    - For each team the player was on: record (min_game_date, max_game_date) = the date range the player was demonstrably on that team
    - Return dict mapping team_id (looked up from teams table by abbreviation) to (start_date, end_date)
    - This handles mid-season trades: player has games with Team A from Oct-Dec and Team B from Jan-Mar

    Function `synthesize_dnp_rows(conn: sqlite3.Connection, season: str) -> int`:
    - Step 1: Get all (player_id, team_id) pairs from team_rosters for this season
    - Step 2: For each player, determine their team tenure using _get_player_team_tenure()
    - Step 3: For each (player_id, team_id) with a known tenure:
      - Query team_schedules for that team's games within the player's tenure date range
      - Cross-reference with player_game_logs: find game_ids where player has NO entry
      - These gaps are the DNP games
    - Step 4: Build DNP rows as list of dicts, each with:
      - player_id, game_id, season, game_date, matchup (from team_schedules)
      - wl=None, min=0, pts=0, reb=0, ast=0, stl=0, blk=0, fg3m=0, fgm=0, fga=0, ftm=0, fta=0, oreb=0, dreb=0, tov=0, pf=0, plus_minus=0
      - is_dnp=1
    - Step 5: Bulk insert using pd.DataFrame(dnp_rows) and INSERT OR IGNORE (via queries.insert_game_logs or direct executemany with INSERT OR IGNORE to handle idempotency)
    - Log: `logger.info(f"Synthesized {len(dnp_rows)} DNP rows for season {season}")`
    - Return len(dnp_rows)

    **Alternative simpler approach if _get_player_team_tenure is too complex:**
    Use a SQL-first approach:
    ```sql
    SELECT tr.player_id, ts.game_id, ts.game_date, ts.matchup, tr.team_id
    FROM team_rosters tr
    JOIN team_schedules ts ON tr.team_id = ts.team_id AND tr.season = ts.season
    WHERE tr.season = ?
      AND NOT EXISTS (
          SELECT 1 FROM player_game_logs pgl
          WHERE pgl.player_id = tr.player_id AND pgl.game_id = ts.game_id
      )
      AND EXISTS (
          SELECT 1 FROM player_game_logs pgl2
          WHERE pgl2.player_id = tr.player_id
            AND pgl2.season = tr.season
            AND pgl2.matchup LIKE '%' || (SELECT abbreviation FROM teams WHERE team_id = tr.team_id) || '%'
            AND pgl2.game_date <= ts.game_date
      )
    ```
    The nested EXISTS ensures the player actually played for that team at some point before or on the game date — preventing false DNP rows for traded players.

    Function `synthesize_all_dnp_rows(conn: sqlite3.Connection, seasons: list[str] = None) -> dict`:
    - Default seasons to SEASONS if None
    - For each season: count = synthesize_dnp_rows(conn, season)
    - Return {"total_dnp_rows": total, "per_season": {season: count, ...}}

    Function `validate_completeness(conn) -> bool`:
    - Count and log:
      - `SELECT COUNT(*) FROM players` → "Players: {n}"
      - `SELECT COUNT(*) FROM teams` → "Teams: {n}"
      - `SELECT COUNT(*) FROM player_game_logs WHERE is_dnp = 0` → "Game logs (real): {n}"
      - `SELECT COUNT(*) FROM player_game_logs WHERE is_dnp = 1` → "Game logs (DNP): {n}"
      - `SELECT COUNT(*) FROM team_stats` → "Team stats: {n}"
      - `SELECT COUNT(*) FROM team_rosters` → "Roster entries: {n}"
      - `SELECT COUNT(*) FROM team_schedules` → "Schedule entries: {n}"
      - Per-season game log counts: `SELECT season, COUNT(*) FROM player_game_logs GROUP BY season ORDER BY season`
      - Collection progress: `SELECT status, COUNT(*) FROM collection_progress GROUP BY status`
    - If any 'failed' entries exist: `SELECT entity_type, entity_id, season, error_message FROM collection_progress WHERE status = 'failed' LIMIT 20` → log each as warning
    - Validation checks (return False if any fail):
      - teams count should be 30 (NBA teams)
      - players count should be >= 400 (per D-02: 450+ active players)
      - team_stats count should be >= 100 (30 teams × 5 seasons = 150, allow some failures)
      - game logs (real) count should be >= 50000 (450 players × ~80 games × ~3 seasons on average)
    - Return True if all checks pass

    Function `_get_current_season() -> str`:
    - Same logic as V1's get_season() but returns single string:
    - If month >= 10: f"{year}-{str(year+1)[-2:]}"
    - Else: f"{year-1}-{str(year)[-2:]}"

    - Parse arguments with argparse:

    - **--validate mode:**
      - Call validate_completeness(conn)

    - **--full mode (per D-01: 5 seasons, per D-02: all active players):**
      - Log: "Starting full data collection for seasons: {SEASONS}"
      - start_time = time.time()
      - Create NBAClient: `client = NBAClient(cache_path=CACHE_PATH)`

      Collection order (dependencies per research):
      1. **Seed teams** (static data, no API call):
         ```python
         teams = client.get_all_teams()
         for t in teams:
             queries.upsert_team(conn, t["id"], t["abbreviation"], t["full_name"])

         ```

      2. **Seed players** (static data, no API call):
         ```python
         players = client.get_all_active_players()

             queries.upsert_player(conn, p["id"], p["full_name"], True)
         conn.commit()
         logger.info(f"Seeded {len(players)} active players")
         ```

      3. **Collect team rosters:** `result = collect_team_rosters(client, conn)`
         Log result counts.

      4. **Collect team schedules:** `result = collect_team_schedules(client, conn)`
         Log result counts.

      5. **Collect team advanced stats:** `result = collect_team_stats(client, conn)`
         Log result counts.
